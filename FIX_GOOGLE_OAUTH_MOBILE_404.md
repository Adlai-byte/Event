# Fix Google OAuth 404 Error on Mobile

## Problem
When logging in with Google on mobile, you get a 404 error. However, it works when opening directly in Chrome.

## Root Cause
The 404 error occurs because the redirect URI generated for mobile OAuth doesn't match what's configured in Google Cloud Console. The redirect URI format differs between:
- Expo Go (development): `exp://127.0.0.1:8081/--/oauth` or `exp://192.168.x.x:8081/--/oauth`
- Standalone app: `com.event://oauth`
- Expo proxy: Uses a different format

## Solution

### Step 1: Get Your Redirect URI
When you run the app and try to login with Google, check the console logs. You'll see output like:
```
Redirect URI (with proxy): https://auth.expo.io/@your-username/Event
Redirect URI (no proxy): com.event://oauth
```

### Step 2: Add All Redirect URIs to Google Cloud Console

1. Go to [Google Cloud Console](https://console.cloud.google.com/)
2. Select your project (e.g., `e-vent-aa93e`)
3. Navigate to **APIs & Services** → **Credentials**
4. Find your **OAuth 2.0 Client ID** (the Web Client ID)
5. Click **Edit**
6. Under **Authorized redirect URIs**, add ALL of these:

```
com.event://oauth
exp://127.0.0.1:8081/--/oauth
exp://192.168.1.100:8081/--/oauth
https://auth.expo.io/@your-username/Event
```

**Important Notes:**
- Replace `192.168.1.100` with your actual local network IP address
- Replace `@your-username/Event` with your actual Expo username and project slug
- The `exp://` URIs are for Expo Go development
- The `com.event://` URI is for standalone builds
- The `https://auth.expo.io/` URI is for Expo proxy (used when `useProxy: true`)

### Step 3: Find Your Exact Redirect URI

To find the exact redirect URI being used:

1. Run your app: `npx expo start`
2. Try to login with Google
3. Check the console output - it will show the exact redirect URI
4. Copy that exact URI and add it to Google Cloud Console

### Step 4: Alternative - Use Direct Scheme (No Proxy)

If you want to use the direct scheme without Expo proxy, you can modify `mvc/services/GoogleWebAuth.ts`:

```typescript
const redirectUri = AuthSession.makeRedirectUri({
  scheme: 'com.event',
  path: 'oauth',
  useProxy: false, // Change to false
});
```

Then make sure `com.event://oauth` is added to Google Cloud Console.

### Step 5: For Production/Standalone Builds

For standalone builds (not Expo Go), you only need:
- `com.event://oauth`

Make sure this is added to your Google Cloud Console OAuth client.

## Troubleshooting

### Still Getting 404?
1. **Check the console logs** - The exact redirect URI is logged when you try to login
2. **Verify in Google Cloud Console** - Make sure the redirect URI matches exactly (including protocol, path, etc.)
3. **Clear app cache** - Try `npx expo start --clear`
4. **Check app.json** - Verify the scheme is `com.event`:
   ```json
   {
     "expo": {
       "scheme": "com.event"
     }
   }
   ```

### Error: "redirect_uri_mismatch"
- The redirect URI in the error message is the one you need to add
- Copy it exactly as shown (including `https://` or `exp://` prefix)
- Add it to Google Cloud Console

### Works in Chrome but not in App
- Chrome uses a different redirect mechanism (web redirect)
- The app uses deep links which require proper configuration
- Make sure the deep link redirect URI is added to Google Cloud Console

## Testing

1. Clear cache: `npx expo start --clear`
2. Try Google login on mobile
3. Check console for the redirect URI being used
4. Verify that exact URI is in Google Cloud Console
5. If still failing, check the error message - it will tell you which redirect URI is expected

## Additional Notes

- The redirect URI format is automatically generated by `expo-auth-session`
- Different environments (Expo Go vs standalone) use different formats
- You may need to add multiple redirect URIs to support all environments
- The `useProxy: true` option uses Expo's proxy service which requires the `https://auth.expo.io/` URI
