# Google OAuth Setup Guide for Mobile

## Problem
Google login works on web but fails on mobile with "Authentication failed" error.

## Solution
This has been fixed by using `expo-auth-session` instead of WebView. However, you need to configure your Google OAuth Client ID.

## Step 1: Get Your Google OAuth Web Client ID

1. Go to [Firebase Console](https://console.firebase.google.com/)
2. Select your project (`e-vent-aa93e` or `authflow-1gvkb`)
3. Navigate to **Authentication** → **Sign-in method**
4. Click on **Google** provider
5. Under **Web SDK configuration**, you'll see:
   - **Web client ID**: This is what you need!
   - Format: `XXXXXXXXXX-XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX.apps.googleusercontent.com`

## Step 2: Configure the Client ID

You have two options:

### Option A: Environment Variable (Recommended)
Create a `.env` file in the root directory (or add to existing one):

```env
EXPO_PUBLIC_GOOGLE_OAUTH_CLIENT_ID=your-web-client-id-here.apps.googleusercontent.com
```

Then restart your Expo development server.

### Option B: Direct Configuration
Edit `mvc/services/GoogleWebAuth.ts` and replace line 18:

```typescript
const GOOGLE_OAUTH_CLIENT_ID = process.env.EXPO_PUBLIC_GOOGLE_OAUTH_CLIENT_ID || 
  'YOUR-ACTUAL-WEB-CLIENT-ID-HERE.apps.googleusercontent.com';
```

## Step 3: Configure Redirect URI in Google Cloud Console

1. Go to [Google Cloud Console](https://console.cloud.google.com/)
2. Select your project
3. Navigate to **APIs & Services** → **Credentials**
4. Find your OAuth 2.0 Client ID (the Web Client ID you copied)
5. Click **Edit**
6. Under **Authorized redirect URIs**, add:
   - `com.event://oauth` (for development)
   - `exp://127.0.0.1:8081/--/oauth` (for Expo Go)
   - `exp://192.168.x.x:8081/--/oauth` (for Expo Go on network, replace with your IP)

**Note:** The redirect URI is automatically generated by `expo-auth-session` based on your app scheme. The format is: `{scheme}://oauth`

## Step 4: Verify Configuration

After setting up, test the Google login on mobile. The console will show:
- The client ID being used
- The redirect URI
- Any errors with detailed messages

## Troubleshooting

### Error: "Invalid OAuth client ID"
- Make sure you're using the **Web Client ID**, not Android or iOS client ID
- Verify the client ID format is correct
- Check that the environment variable is set correctly

### Error: "Redirect URI mismatch"
- Add the redirect URI to Google Cloud Console (Step 3)
- The redirect URI format should match: `com.event://oauth`
- For Expo Go, also add the Expo redirect URIs

### Error: "No ID token received"
- This usually means the token exchange failed
- Check that your OAuth client is properly configured
- Verify you have the correct scopes enabled

## Testing

1. Clear your app cache: `expo start --clear`
2. Try Google login on mobile
3. Check the console logs for detailed error messages
4. The browser should open automatically (not a WebView)

## Additional Notes

- The Web Client ID is different from Android/iOS client IDs
- Make sure you're using the Web Client ID for `expo-auth-session`
- The redirect URI scheme (`com.event`) is configured in `app.json`
